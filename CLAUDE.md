# 작업 규칙 (Gim)

## 소통 규칙 (반드시 준수)

1. **어체**: Gim과의 모든 작업·질문·답변은 항상 **정중하고 예의 있는 존댓말**로 합니다. 반말·편한 말투는 사용하지 않습니다.
2. **설명 방식**: 모든 설명은 **초보자 입장**에서 정리합니다.
   - 전문 용어를 쓸 때는 쉬운 말로 풀어서 함께 설명합니다.
   - 결론을 먼저 말하고, 그다음 이유와 근거를 친절하게 브리핑합니다.
   - 필요하면 비유·예시·표·그림(아티팩트)으로 눈에 보이게 정리합니다.
3. **호칭**: 사용자는 **Gim** 으로 부릅니다.
4. **작업 마무리**: 코드에 영향을 주는 작업이 끝날 때마다 **항상 `docs/play.html`을 재빌드해 Gim에게 띄워(전송)** 줍니다. (코드 변경이 없는 순수 문서·분석 작업은 예외.)

## 프로젝트 개요

- **엘드리아 연대기** — 방치형 수집 RPG (React Native / Expo, react-native-web).
- 웹 빌드: `npm run build:play` → `docs/play.html` (단일 파일).
- 구조: `system/core/` = 규칙·계산(모듈식, 기능별 `.mjs`) · `app/` = 화면·렌더. 코어는 화면을 참조하지 않는 단방향 의존.
- 아트 연결점: `app/charImages.js` (캐릭터 이미지 단일 등록 지점).

## 코드 작성 원칙

- 기능은 모듈 단위로 나눕니다(한 기능 = 한 모듈).
- 재화·밸런스 등 공통 규칙은 기반 모듈(예: `economy`)로 모아 공유하고, 의존은 기반→기능 한 방향으로 유지합니다.
- 테스트: `node --test system/test/*.test.mjs`.
