# IP 게임 시스템 설계서

## 0. 한 줄 정의

> **IP = 게임 시스템(엔진) 그 자체.**
> 시스템은 고정 자산이고, 그 위에 **장르(RPG↔방치형)** 와 **컨셉(테마)** 만 갈아끼워
> 여러 게임으로 운영한다.

캐릭터·세계관이 IP가 아니다. **성장·전투·경제 규칙**이 IP다.
따라서 한 번 잘 만든 시스템을 판타지 RPG로도, SF 방치형으로도 재출시할 수 있다.

---

## 1. 3-레이어 아키텍처

```
┌─ Core System (IP · 고정) ───────────────────────────┐
│  장르도 컨셉도 모른다. 순수 규칙과 데이터.               │
│    · archetypes  유닛 역할(방어/공격/지원)              │
│    · stats       레벨·랭크 성장 공식                    │
│    · units       유닛 인스턴스 → 전투 프로필              │
│    · resolution  전투 판정 엔진  → { win, duration }    │
│    · progression 무한 스테이지 사다리                    │
│    · economy     추상 자원(currency/growth/summon)     │
│    · gameState   세이브(보유·진행·지갑) — 장르/컨셉 없음  │
├─ Genre Layer (Driver · 스왑) ───────────────────────┤
│  "시간을 누가 굴리나" 만 다르다. 같은 resolve()를 쓴다.   │
│    · rpg   플레이어가 battle() 트리거, 행동력 소모, 능동   │
│    · idle  tick(dt) 자동 반복·누적, 오프라인·환생, 수동0  │
├─ Concept Layer (Skin · 스왑) ───────────────────────┤
│  숫자는 그대로. 이름·이모지·색상만 매핑.                  │
│    · fantasy  VANGUARD→수호기사, currency→골드           │
│    · scifi    VANGUARD→가디언프레임, currency→크레딧       │
└─────────────────────────────────────────────────────┘
```

의존 방향은 **한 방향뿐**: `concept → genre → core`.
core 는 위 레이어를 절대 import 하지 않는다 → 그래서 재사용된다.

## 2. 핵심 설계: 판정 엔진이 두 장르를 잇는다

`resolution.resolve(party, challenge)` 는 결정론적으로 다음을 반환한다:

```js
{ win: boolean, duration: number /* 클리어 소요 초 */ }
```

- **RPG** 는 `win` 을 쓴다 → 이기면 다음 스테이지 개방(진행 게이팅).
- **방치형** 은 `duration` 을 쓴다 → `dt / duration` 으로 초당 클리어 수·보상 누적.

> 하나의 순수 함수가 능동형과 자동형을 동시에 지탱한다. 이게 이 IP의 심장이다.

판정 모델(요약): 각 진영을 `HP · 유효DPS` 로 환산해 상호 처치 시간을 비교.
방어(def)는 `100/(100+def)` 피해 감쇠, 지원형은 팀 공격력 버프를 더한다.

## 3. 레이어별 스왑 규칙

| 바꾸는 것 | 건드리는 파일 | core 변경 | 예시 |
|---|---|---|---|
| **장르** | `genres/*.mjs` 추가/교체 | ❌ 없음 | RPG → 방치형 |
| **컨셉** | `concepts/*.mjs` 추가/교체 | ❌ 없음 | 판타지 → SF |
| **밸런스** | `core/*` 상수 | ⭕ 공통 반영 | 성장률, 적 강화율 |

새 장르(예: 실시간 PvP)나 새 컨셉(예: 좀비 아포칼립스)을 붙일 때
**core 는 한 줄도 수정하지 않는다.** 파일 하나만 추가하면 된다.

---

## 4. 검증된 동작 (`node system/demo.mjs`)

같은 IP 상태 하나로:

1. **컨셉 스왑** — 판타지/SF 로 렌더. 스탯 숫자 **완전 동일**, 이름만 다름.
2. **장르 스왑**
   - RPG: 3회 능동 전투 → 4층 개방, 행동력 소모
   - 방치형: 2시간 오프라인 → 벽(7섹터)까지 자동 전진 후 파밍, 보상 누적, 환생

## 5. 폴더 구조

```
system/
├── core/            # IP — 고정 자산 (장르/컨셉 무관)
│   ├── archetypes.mjs
│   ├── stats.mjs
│   ├── units.mjs
│   ├── resolution.mjs   ← 판정 엔진 (심장)
│   ├── progression.mjs
│   ├── economy.mjs
│   └── gameState.mjs
├── genres/          # Driver — 스왑
│   ├── rpg.mjs
│   └── idle.mjs
├── concepts/        # Skin — 스왑
│   ├── fantasy.mjs
│   ├── scifi.mjs
│   └── index.mjs
└── demo.mjs         # 스왑 증명 데모
```
