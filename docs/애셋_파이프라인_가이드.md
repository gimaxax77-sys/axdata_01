# 애셋 파이프라인 가이드 — 새 캐릭터·새 게임 만들기

이 게임은 **"시스템(IP)은 고정, 컨셉(스킨)만 갈아끼운다"**는 구조입니다.
그래서 아트도 **로스터 데이터를 원천으로 자동 생성**하도록 다리(bridge) 스크립트를
붙여 두었습니다. 이 문서는 그 다리를 쓰는 법을 정리합니다.

> 아트 스튜디오(`axdata_09`)를 켜고, 게임에서 명령 한 줄이면 그림이 채워집니다.
> API 키 설정은 아트 스튜디오의 `docs/API키_설정_가이드.md`를 참고하세요.

---

## 다리 스크립트 두 개

| 명령 | 하는 일 | 결과물 | 갱신하는 레지스트리 |
| --- | --- | --- | --- |
| `npm run sync-art` | 캐릭터 **초상** 생성 | `assets/char/<컨셉>/<id>.png` (512+ 투명) | `app/charImages.js` |
| `npm run sync-sprites` | **전투 스프라이트**(idle/attack) 생성 | `assets/units/<컨셉>/<id>/<id>_<상태>.png` | `app/unitSprites.js` |

두 스크립트 모두 **컨셉을 가리지 않고** `system/concepts`의 모든 컨셉을 순회합니다.
이미 있는 파일은 건너뛰므로(캐시), 부족한 것만 채워집니다.

### 실행 전 준비
아트 스튜디오 서버를 먼저 켭니다(기본 `http://127.0.0.1:8000`).
```bash
# axdata_09 에서
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```
그런 다음 게임에서 실행합니다.
```bash
# axdata_01 에서
ART_API=http://127.0.0.1:8000 npm run sync-art
ART_API=http://127.0.0.1:8000 npm run sync-sprites
```

### 자주 쓰는 옵션
```bash
npm run sync-art -- --concept fantasy   # 한 컨셉만
npm run sync-art -- --limit 3           # 앞에서 N개만(테스트용)
npm run sync-art -- --force             # 있어도 다시 생성
npm run sync-art -- --map-only          # 생성 없이 레지스트리만 갱신
```

## 시나리오 A — 새 캐릭터 추가 (기존 게임에 콘텐츠 붓기)

1. **로스터에 한 줄 추가.** `system/concepts/fantasy.mjs`(또는 `scifi.mjs`)의 `roster`에
   새 캐릭터를 넣습니다. 기존 항목을 복사해 값만 바꾸면 됩니다.
   ```js
   { id: 'newhero', name: '새영웅', emoji: '⭐', title: '별의 기사',
     personality: '용감한', element: 'LIGHT', archetype: 'STRIKER',
     rarity: 'SR', signature: 'SIG_STAR_EDGE',
     lines: { greet: '...', bond: '...', levelup: '...' } },
   ```
2. **그림 생성.**
   ```bash
   npm run sync-art && npm run sync-sprites
   ```
3. 끝. `charImages.js`·`unitSprites.js`가 자동 갱신되어 로스터·소환·전투에 바로 반영됩니다.

> 그림이 아직 없으면 게임은 **이모지로 자연 폴백**하므로, 데이터만 먼저 넣고 그림은
> 나중에 채워도 됩니다.

## 시나리오 B — 새 게임(컨셉) 추가 (스킨 갈아끼우기)

이 게임의 핵심입니다. **컨셉 파일 하나를 추가하면, 같은 시스템이 전혀 다른 게임이 됩니다.**
`system/concepts/wuxia.mjs`(무협)가 그 실제 예시로 들어 있습니다.

1. **컨셉 파일 작성.** `system/concepts/scifi.mjs`를 복사해 새 파일을 만듭니다.
   - `id`, `title`, `palette`만 새 정체성으로.
   - `archetypes`·`resources`·`terms`·`elements`의 **이름/이모지만** 컨셉에 맞게 바꿉니다.
   - `roster`는 **id·element·archetype·rarity·signature를 그대로 두고**
     `name`·`title`·`personality`·`emoji`·`lines`만 새 컨셉 말투로 바꿉니다.
     (같은 캐릭터 슬롯, 다른 인물 — 이래야 시스템·밸런스가 그대로 재사용됩니다.)
   - `costumes`·`campaign`도 같은 방식(구조 유지, 표현만 교체).
2. **컨셉 등록.** `system/concepts/index.mjs`의 `CONCEPTS`에 한 줄 추가합니다.
   ```js
   import { wuxiaConcept } from './wuxia.mjs';
   export const CONCEPTS = { fantasy, scifi, wuxia };
   ```
3. **아트 매핑(선택).** 다리(`scripts/sync-art.mjs`, `sync-sprites.mjs`)의 `GENRE`/`STYLE`
   맵에 새 컨셉의 장르/아트 스타일을 한 줄 추가하면 실제 아트 품질이 좋아집니다.
   (없어도 컨셉 id가 그대로 쓰여 동작은 합니다.)
   ```js
   const GENRE = { fantasy: 'high fantasy', scifi: 'sci-fi cyberpunk', wuxia: 'wuxia martial arts' };
   ```
4. **그림 생성.**
   ```bash
   npm run sync-art -- --concept wuxia
   npm run sync-sprites -- --concept wuxia
   ```
5. 끝. 새 컨셉의 캐릭터 그림이 `assets/char/wuxia/`에 자동으로 채워지고 레지스트리에
   등록됩니다. 빌드 시 컨셉만 골라 서로 다른 게임으로 출시할 수 있습니다.

## 요약 그림

```
system/concepts/  ┌ fantasy  → 게임1 (엘드리아)
 (스킨 추가 지점)  ├ scifi    → 게임2 (오비탈 프로토콜)
                  └ wuxia    → 게임3 (무협) ...
        │  새 파일 하나 추가하면
        ▼
 sync-art / sync-sprites  (컨셉 무관 · 그대로 재사용)
        ▼
 아트 스튜디오(axdata_09)  →  assets/ 에 그림 자동 채움
```

**게임 하나 = 컨셉 파일 하나 + 다리 재실행.** 이것이 이 프로젝트의 확장 방식입니다.
